#### 接口中的abstract, default, staitc三类方法的关系
1. abstract方法在接口中是只有方法签名的方法，需要接口的实现类提供实现
2. default方法是接口拥有实现体，且被所有实现类共享的方法，任何实现类的实例都可以调用
3. static是独属于接口的方法，于接口的所有实现类无关，可通过接口名调用

| 异同 | abstract | default | static |
| --------------- | --------------- | --------------- | --------------- |
| 实现 | 否 | 是 | 是 |
| 重写 | 是 | 是 | 否 |
| 调用方 | 实现类实例 | 实现类实例 | 接口 |

#### 眚写机制在确 定目标时，返回值是否会影响
有的。在java的重写机制中，首先会判断新方法与目标方法的方法签名是否一致。在一致的前提下，在判断它们的返回类型是否相等或满足继承关系。以上条件全部满足，则重写成功，反之，报错。

#### Java为什么在重写机制中检查访问修饰符与异常是否兼容
进行访问权限检查与异常检查的目的为了避免出现在多态机制下定义行为与实际行为之间的矛盾。
